## v2.0.0 — ترقية محرك الواتساب + صندوق الردود المباشر
**التاريخ:** 2026-04-04

### ملخص تنفيذي
تم ترقية البنية التحتية لإرسال الواتساب بالكامل من مكتبة `whatsapp-web.js` إلى `WPPConnect` لتحقيق استقرار أعلى وحماية متقدمة من الحظر. كما تم إضافة ميزة **صندوق الردود المباشر** لاستقبال والرد على رسائل العملاء فورياً.

---

### التحديثات الرئيسية

#### 1. ترقية محرك الواتساب (WPPConnect)
- **ما تغير:** استبدال `whatsapp-web.js` بـ `@wppconnect-team/wppconnect`
- **الفائدة للعميل:** جلسة واتساب أكثر استقراراً — لا حاجة لمسح الباركود عند كل إعادة تشغيل للسيرفر
- **تفاصيل تقنية:**
  - حفظ التوكنات تلقائياً في مجلد `storage/tenant_<id>/wpp_tokens/`
  - مزامنة حالة الجلسة مع قاعدة البيانات (أعمدة جديدة: `whatsapp_status`, `whatsapp_phone`)
  - إشعارات فورية عبر Socket.io لجميع تغييرات الحالة

#### 2. نظام مكافحة الحظر المتقدم (Anti-Ban Engine v2)
- **ما تغير:** ترقية من تأخير عشوائي بسيط إلى 6 طبقات حماية ذكية
- **الفائدة للعميل:** حماية أقوى بكثير من حظر واتساب أثناء الحملات
- **الطبقات:**
  | الطبقة | الوصف |
  |--------|-------|
  | توزيع غاوسي | تأخير طبيعي بتوزيع جرسي بدل العشوائية المسطحة |
  | محاكاة الكتابة | مؤشر "يكتب..." يظهر بمدة تتناسب مع طول الرسالة (50ms لكل حرف، حد أقصى 3 ثواني) |
  | مراعاة الوقت | تباطؤ تلقائي في ساعات الليل والفجر |
  | إحماء الجلسة | أول 8 رسائل تُرسل ببطء أكبر |
  | حارس الحد اليومي | تباطؤ تدريجي بعد 150 رسالة/يوم |
  | استراحات عشوائية | فترات راحة 3-8 دقائق بنسبة 7% من الرسائل |

- **تسلسل إرسال كل رسالة:**
  1. `checkNumberStatus()` — التحقق من وجود الرقم على واتساب
  2. `startTyping()` → تأخير حسب طول الرسالة → `stopTyping()`
  3. `sendText()` / `sendImage()` — الإرسال الفعلي
  4. تأخير بين الرسائل (30-60 ثانية، توزيع غاوسي)

#### 3. صندوق الردود المباشر (Live Inbox) — ميزة جديدة
- **ما تغير:** صفحة جديدة بالكامل لاستقبال والرد على رسائل الواتساب
- **الفائدة للعميل:** يمكنه الآن رؤية ردود المدعوين والتفاعل معهم مباشرة من لوحة التحكم
- **المكونات:**
  - تصميم ثنائي الأجزاء (قائمة المحادثات + نافذة الدردشة)
  - رسائل فورية عبر Socket.io — لا حاجة لتحديث الصفحة
  - حفظ تاريخ المحادثات في قاعدة البيانات (جدول `messages`)
  - إشارة قراءة تلقائية (`sendSeen`)
  - شارة إشعارات حمراء في القائمة الجانبية
  - بحث في المحادثات
  - نقطة خضراء نابضة للرسائل الجديدة

### الملفات المعدلة
| الملف | نوع التعديل |
|-------|-------------|
| `package.json` | استبدال whatsapp-web.js بـ wppconnect |
| `src/core/WhatsAppManager.js` | إعادة كتابة كاملة — WPPConnect + onMessage |
| `src/core/AntiBanEngine.js` | إعادة كتابة كاملة — 6 طبقات حماية |
| `src/core/processBatch.js` | تحديث تسلسل الإرسال + محاكاة الكتابة |
| `src/config/settings.js` | تحديث التأخير إلى 30-60 ثانية |
| `src/server.js` | إضافة مسارات صندوق الردود + API |
| `src/database/init_saas.js` | إضافة جدول messages |
| `src/database/migrate_saas.js` | إضافة migration لـ messages + أعمدة واتساب |
| `src/views/dashboard/inbox.ejs` | صفحة صندوق الردود (جديدة) |
| `src/views/partials/sidebar.ejs` | رابط صندوق الردود في القائمة |

### خطوات التشغيل بعد التحديث
```bash
# 1. تثبيت المكتبة الجديدة
npm install

# 2. تشغيل الترحيل لإنشاء جدول messages
npm run db:migrate

# 3. إعادة تشغيل السيرفر
npm start
```
